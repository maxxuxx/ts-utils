# Electron updater module

[한국어](./readme.kr.md)

Reusable Electron update service, preload bridge, state utilities, channel naming, and electron-builder publish config helpers.

## Use this when

- An Electron main process needs a small service around `electron-updater` state and IPC.
- A preload script needs to expose update actions and state subscriptions to the renderer.
- Build scripts need repeatable generic, S3, or GitHub publish config generation.

## Import

```ts
import { createUpdaterService, registerUpdaterIpcHandlers } from "@maxxuxx/ts-utils/electron-updater/main";
import { createUpdaterBridge } from "@maxxuxx/ts-utils/electron-updater/preload";
import { createPublishConfig } from "@maxxuxx/ts-utils/electron-updater/builder";
import { resolveUpdaterChannels } from "@maxxuxx/ts-utils/electron-updater";
```

## Core exports

| Export | Role |
|---|---|
| `electron-updater/main` | Creates the updater service and registers main-process IPC handlers. |
| `electron-updater/preload` | Creates or exposes the renderer updater bridge. |
| `electron-updater/builder` | Creates electron-builder publish config and patches updater cache names. |
| `resolveUpdaterChannels` | Resolves default or custom IPC channel names. |
| `createInitialUpdateState`, `createUpdateStateResolver`, `clampProgressPercent` | Normalize updater state and progress values. |
| `appUpdateStateSchema` | Zod schema for bridge payload validation. |

## Basic example

```ts
const updater = createUpdaterService({
  app,
  autoUpdater,
  feedUrl: "https://updates.example.com/desktop",
  getWindow: () => mainWindow,
  logger,
  autoInstallOnDownloaded: true
});

updater.setup();

registerUpdaterIpcHandlers({
  ipcMain,
  service: updater
});
```

## Behavior notes

- `electron` and `electron-updater` are optional peers. The consuming app owns the runtime versions.
- `feedUrl` is optional when packaged builds use the `app-update.yml` generated by `electron-builder`.
- By default, unpackaged apps are disabled with reason `not-packaged`.
- The bridge exposes `check`, `start`, `install`, `getState`, and `onState`.

## Edge cases

- `requireFeedUrl` disables the service when no feed URL is configured.
- `check` temporarily disables auto-download so it only checks availability.
- `check` keeps a previously discovered `availableVersion` while the status is `checking`.
- `start` downloads an available update or checks with auto-download enabled.
- `createGithubPublishConfig` (and `createPublishConfig`'s `github` entry) is skipped unless both `owner` and `repo` are provided.
- Preload bridge state payloads are parsed with Zod before reaching renderer listeners.
