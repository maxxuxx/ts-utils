# Electron updater module

Reusable Electron update helpers for `electron-updater`

The module keeps Electron dependencies injectable so apps own their Electron runtime versions

```bash
npm install @maxxuxx/ts-utils electron-updater
npm install -D electron electron-builder
```

`electron` and `electron-updater` are optional peer dependencies of `@maxxuxx/ts-utils`

## Main process

```ts
import { app, ipcMain } from "electron";
import electronUpdater from "electron-updater";
import {
  createUpdaterService,
  registerUpdaterIpcHandlers
} from "@maxxuxx/ts-utils/electron-updater/main";

const { autoUpdater } = electronUpdater;

const updater = createUpdaterService({
  app,
  autoUpdater,
  feedUrl: appConfig.updateFeedUrl,
  getWindow: getMainWindow,
  logger: log,
  autoInstallOnDownloaded: true
});

updater.setup();

registerUpdaterIpcHandlers({
  ipcMain,
  service: updater
});
```

`feedUrl` is optional. Omit it when the packaged app should use the `app-update.yml` generated by `electron-builder`.

## Preload

```ts
import { contextBridge, ipcRenderer } from "electron";
import { createUpdaterBridge } from "@maxxuxx/ts-utils/electron-updater/preload";

const updater = createUpdaterBridge(ipcRenderer);

contextBridge.exposeInMainWorld("appApi", {
  updater
});
```

The bridge exposes `check`, `start`, `install`, `getState`, and `onState`.

## Renderer

```ts
const state = await window.appApi.updater.check();

if (state.status === "available") {
  await window.appApi.updater.start();
}

const dispose = window.appApi.updater.onState((nextState) => {
  console.log(nextState.status, nextState.progressPercent);
});
```

## electron-builder publish config

```ts
import {
  applyAwsCredentials,
  createPublishConfig,
  patchUpdaterCacheDirName
} from "@maxxuxx/ts-utils/electron-updater/builder";

applyAwsCredentials();

export default {
  publish: createPublishConfig({
    generic: {
      url: process.env.APP_UPDATE_BASE_URL
    },
    s3: {
      bucket: process.env.APP_UPDATE_S3_BUCKET,
      region: process.env.APP_UPDATE_S3_REGION,
      path: process.env.APP_UPDATE_S3_PATH
    }
  }),
  afterSign: async (context) => {
    // read app-update.yml, call patchUpdaterCacheDirName, and write it back
  }
};
```

Use S3 or CloudFront for the app update feed. GitHub publish config is available for artifact publishing, but private GitHub update feeds require client-side GitHub credentials and are usually not a good default for private desktop apps.
