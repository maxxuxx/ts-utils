# ts-utils 0.8 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver `@maxxuxx/ts-utils@0.8.0` with Electron removed, unsafe defaults corrected, deterministic edge-case behavior, shared Result and base64url primitives, and package-level release verification

**Architecture:** Keep the empty package root and existing feature subpaths while introducing only `result` and `encoding/base64url`. Build shared primitives first, apply them to auth and runtime modules through focused TDD cycles, then update package metadata, documentation, CI, and migration guidance

**Tech Stack:** TypeScript 5.9, NodeNext ESM, Node 22.12 and 24, Vitest 4, Zod 4, React 18 and 19 compatible types, iron-session 8

## Global Constraints

- Target package version is `0.8.0`
- Electron source, exports, dependencies, tests, and documentation stay removed
- Package root remains intentionally empty
- Browser and server subpath imports remain the public consumption model
- Runtime typing is trusted only after schema validation
- Authentication material, query strings, bodies, and raw upstream text never appear in public errors
- Existing public names remain only when they preserve the new safe behavior
- Every behavior change follows red, green, refactor with the failing output observed first
- Update `agent.md`, `readme.md`, and `readme.kr.md` for every changed module
- Preserve all current uncommitted Electron-removal work
- Do not publish or push during this plan
